Audit the rules before you trade.

BetAudit is a pre-trade resolution auditor for autonomous prediction-market agents. Give it a Polymarket market URL; it reads the market's real resolution rules, oracle metadata, and dispute terms, and returns a 0–100 resolution risk score with a machine-actionable verdict — PROCEED, CAUTION, or ABORT_TRADE — so a bot never buys a market a hidden clause would trap.

POLYMARKET · UMA OKX A2MCP · AGENT #6141 REST + MCP

01Quickstart

Two calls: mint a demo key, then verify a market. The demo key route is public and rate-limited.

# 1. mint a demo API key
curl -X POST https://betauditmcp.xyz/demo/key
# -> { "api_key": "rl_live_...", ... }  (shown once)

# 2. verify a market
curl -X POST https://betauditmcp.xyz/verify-resolution-rules \
  -H "X-API-Key: rl_live_..." \
  -H "content-type: application/json" \
  -d '{"market_url":"https://polymarket.com/market/<slug>","queried_side":"YES"}'

Prefer to click? Open the live console and paste a market URL — it auto-mints a key for you.

Request

  "market_url": "https://polymarket.com/market/<slug>",  // required

Response

{
  "resolution_risk_score": 85,             // 0-100
  "action": "ABORT_TRADE",               // PROCEED | CAUTION | ABORT_TRADE
  "parsed_contract_data": {
    "oracle_type": "UMA_OOv2",
    "challenge_window_hours": 2.0,
    "source_of_truth_specified": "Official SEC 8-K Filing",
    "current_oracle_state": "UNRESOLVED"
  },
  "rule_mismatches_detected": [
    { "clause": "...", "trader_thesis": "...", "conflict_reason": "..." }
  ],
  "confidence": 0.9,
  "summary": "ABORT_TRADE: resolution hinges on 'Official SEC 8-K Filing', not the headline."
  // + score_breakdown, citations, monitor, request_id, checked_at, schema_version
}

What the fields mean

  • resolution_risk_score — how likely a naive headline trade is trapped (0 = rules match the obvious reading, 100 = the rules betray a headline trader).
  • action — the three-way verdict an agent acts on. ≥70 aborts, ≥35 cautions, else proceeds.
  • parsed_contract_data — the real oracle metadata read from the market.
  • rule_mismatches_detected — concrete gaps between the binding rules and the naive thesis, grounded in the resolution text.

03MCP (for OKX agents)

BetAudit is registered on the OKX AI Marketplace as an A2MCP service (Agent #6141). Agents call it over the Model Context Protocol at POST /mcp (JSON-RPC 2.0, Streamable HTTP) — initialize, tools/list, and tools/call. It exposes one tool, verify_resolution_rules, backed by the same engine as the REST API. 

# list tools
curl -X POST https://betauditmcp.xyz/mcp -H "content-type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"tools/list"}'

# call the tool
curl -X POST https://betauditmcp.xyz/mcp -H "content-type: application/json" \
  -d '{"jsonrpc":"2.0","id":2,"method":"tools/call","params":{
       "name":"verify_resolution_rules",
       "arguments":{"market_url":"https://polymarket.com/market/<slug>","queried_side":"YES"}}}'

The tool returns a human-readable summary plus structuredContent (the full response above).

04How the score is built

  • Read the real rules — resolves the market via Polymarket's Gamma API for the resolution criteria, UMA oracle type, challenge window, and live oracle state.
  • Audit the clauses — an LLM grounds strictly in the resolution text (temperature 0, forced structured output) to surface concrete rule mismatches. With no LLM key it falls back to a deterministic metadata rubric.
  • Score the gap — every point traces to a named component in score_breakdown, so the number is always explainable.
